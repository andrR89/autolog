# Internationalization — AutoLog

Setup de i18n para EN/ES (Onda 4 — item #16). PT-BR continua sendo o
template (`app_pt.arb`); EN e ES já têm traduções iniciais, mas vão
precisar de revisão por falante nativo antes do lançamento.

## Estrutura

```
lib/l10n/
  app_pt.arb           ← template + idioma default
  app_en.arb           ← inglês
  app_es.arb           ← espanhol
  generated/           ← gerado por `flutter gen-l10n`, NÃO editar manualmente
    app_localizations.dart
    app_localizations_pt.dart
    app_localizations_en.dart
    app_localizations_es.dart
```

Config em `l10n.yaml` (raiz do projeto). Generation automática quando
`flutter pub get` rodar (`generate: true` no pubspec).

## Como adicionar uma string nova

1. Adicionar chave + valor em **`app_pt.arb`** (template).
2. Adicionar a mesma chave em **`app_en.arb`** e **`app_es.arb`** com a tradução.
3. Rodar `flutter gen-l10n` (ou só `flutter pub get`).
4. Usar no widget:

```dart
import 'package:autolog/l10n/generated/app_localizations.dart';

final l10n = AppLocalizations.of(context);
Text(l10n.minhaNovaChave);
```

## Como o usuário troca o idioma

Settings → **Idioma** → bottom sheet com:
- Padrão do sistema (null no provider)
- Português / English / Español

Persistido em `SharedPreferences` (`locale_code`). Como `MaterialApp.router`
recebe `locale:` reativo via Riverpod, a UI re-renderiza imediatamente.

## Locales suportados

Definido em `lib/features/settings/locale_providers.dart`:

```dart
const supportedLocales = <Locale>[
  Locale('pt'),
  Locale('en'),
  Locale('es'),
];
```

Quando adicionar novo idioma:
1. Criar `app_xx.arb`
2. Adicionar `Locale('xx')` aqui
3. Adicionar caso em `localeDisplayName`
